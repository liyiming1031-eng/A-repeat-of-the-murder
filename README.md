# 🔦 罪案推理 · 侦探笔记

一个以**真实历史背景为蓝本改编**的纯前端推理探案游戏。勘查线索、推敲证词、指认真凶——用逻辑还原每一起谜案。

> 灵感来源于罪案播客对真实案件的演绎推理。本作的案件内容为面向游戏体验的戏剧化改编，采用公共领域历史背景，**不涉及任何在版权保护期内的播客节目或文学作品的受保护表达**。

## 🎮 玩法

游戏采用「剧本杀」式的分步推理流程：

1. **翻开案件簿** —— 选择一桩待解谜案
2. **案情简报** —— 阅读案发经过与死者档案，认识涉案人员
3. **勘查调查** —— 从四类线索中自由选择查看：
   - 🔍 案发现场
   - 🧪 法医检验
   - 🗣️ 证人证词
   - 📜 文书物证
   - 点击任意嫌疑人可查看其生平、不在场说明、动机与询问记录
4. **推理指认** —— 锁定你心中的真凶，并选出最能坐实结论的「呈堂铁证」
5. **真相揭晓** —— 查看推理还原，获得侦探评分（S / A / B）

评分规则：指认正确且关键证据命中率高 → **S 完美推理**。

## 🛠️ 技术栈

- **纯前端**：HTML + CSS + 原生 JavaScript，零依赖、零构建步骤
- 可直接托管在 GitHub Pages 等任意静态服务上
- 案件数据与游戏引擎分离（`js/cases.js` / `js/game.js`），便于扩展新案件

## 🚀 本地运行

```bash
# 任意静态服务器均可，例如 Python
python -m http.server 8000
# 然后浏览器打开 http://localhost:8000
```

或直接用浏览器打开 `index.html`。

## ➕ 扩展新案件

在 `js/cases.js` 的 `CASES` 数组中新增一个案件对象即可，结构如下：

```js
{
  id: "case-id",
  title: "案件标题",
  subtitle: "年代 · 地点",
  era: "时代 · 地区",
  difficulty: 3,            // 1-5
  duration: "约 15 分钟",
  accent: "#c9a86a",       // 主题色
  synopsis: "一句话简介",
  briefing: { narrative: "案发经过…", victim: { name, age, role, description } },
  suspects: [ { id, name, role, age, portrait, bio, alibi, motive, statements: [] } ],
  clues: [ { id, category, title, locked, detail } ],   // category: scene/forensic/witness/document
  solution: { culpritId, summary, method, keyEvidence: [clueId…], reasoning: [ { step, title, text } ] }
}
```

无需改动任何引擎代码，新案件会自动出现在案件簿中。
